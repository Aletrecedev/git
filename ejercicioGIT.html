Examen Git y Github

  1. ¿Que es un sistema de control de versiones?

    Un sistema de control de versiones es una herramienta que ayuda a los desarrolladores a gestionar cambios en el código fuente a lo largo del tiempo. Permite rastrear modificaciones, revertir cambios a versiones anteriores y trabajar en colaboración sin conflictos.

  2. ¿Cual es la diferencia entre git y github?

    Git: Es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar y controlar el historial de versiones de su código. GitHub: Es una plataforma basada en la web que utiliza Git para el control de versiones, pero ofrece características adicionales como repositorios remotos, colaboración en proyectos, gestión de incidencias, y más.

  3. ¿Cómo creo una rama en git?

    Con el comando "git branch nombre-de-la-rama"

  4. ¿Cómo clono un repositorio?

    Con el comando "git clone URL-del-repositorio"

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Con el comando "git pull"

  6. ¿Cómo puedo gener un tag?

    Con los comandos "git tag -a v1.0 -m "Mensaje del tag" git push origin v1.0"

  7. Cómo hago una mezcla de ramas?

    Con el comando "git merge nombre-de-la-rama"

  8. ¿Cómo puedo moverme a un commit especifico?

    Con el comando "git checkout ID-del-commit"

  9. ¿Cómo puedo borrar una rama?

    Con el comando "git branch -d nombre-de-la-rama"

  10. ¿Cómo puedo borrar un tag?

    Con el comando "git tag -d nombre-del-tag git push origin :refs/tags/nombre-del-tag"

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Con el comando "git status"
